Software Engineering Best Practices

Clean Code

I strongly believe that clean, readable code is the foundation of scalable and maintainable applications. Following principles like simplicity, single responsibility, and meaningful naming improves collaboration, reduces technical debt, and ensures long-term project success.

Pair/Mob Programming

I value collaborative development practices like pair and mob programming for their ability to improve code quality, foster knowledge sharing, and strengthen team dynamics. These methods accelerate problem-solving and create a shared understanding across the team.

Mob Code Review

As a followup practice of Pair/Mob programming, this collaborative approach strengthens collective ownership and reduces time to production.

Test Driven Development

TDD can be challenging to apply effectively on DOM manipulation or code with significant side effects, but when used in the right context, it’s a powerful method. It drives the creation of robust, reliable, and maintainable code while ensuring clear requirements and reducing bugs.

Feature Branch Development

I favor feature branch development for its ability to isolate changes, simplify collaboration, and maintain a clean main branch that always reflect production state. This workflow ensures that new features are thoroughly tested and reviewed before integration, reducing the risk of breaking changes.

Conventional Commits

I follow the Conventional Commits standard to ensure clarity and consistency in commit messages. This practice not only improves collaboration and versioning but also makes the history more readable, helps identify the purpose of each change, and simplifies debugging by tracing code evolution. It also facilitates rollbacks and problem isolation when needed.
